Lec-5: coding interview

  1. make sure what type of questions are expected
  2. is it front side or server side or algorithm
  3. make sure what programming language you would select
  4. be talking while you would be thinking about the solution
  5. Don't give the best solution for the first time...rather walk through that and pretend that you are finding out on the fly
  6. talk about the error handling and tell them that you are avoiding those
  7. don't skim through the code... you will forget or mess up during the actual interview. Rather execute the code... 3 times to make sure it works
  8. most of the interviewer don't care about syntaxtical accuracy. but inform that.
